# 02. Claude Code の使い方

> **参考:** [Claude Code の仕組み — 公式ドキュメント（日本語）](https://code.claude.com/docs/ja/how-claude-code-works)

---

## Claude Code とは

Claude Code は Anthropic が提供する AI コーディングアシスタントの CLI（コマンドラインツール）です。ターミナル上で AI と会話しながら、コードの作成・修正・説明・デバッグを行えます。

単なるコード補完ではなく、**ファイルを読み・編集し・コマンドを実行する**エージェントとして動作します。「このバグを直して」と伝えるだけで、Claude が関連ファイルを探し、コードを理解し、修正し、テストまで行います。

---

## 起動方法

```bash
# プロジェクトフォルダに移動して起動
cd your-project
claude
```

初回起動時は Anthropic アカウントへのサインインが求められます。画面の指示に従ってください。

---

## エージェントループの仕組み

Claude Code は以下の3フェーズを繰り返しながらタスクをこなします。

```
プロンプト入力
    ↓
① コンテキストの収集（ファイルを読む・検索する）
    ↓
② アクションの実行（ファイル編集・コマンド実行）
    ↓
③ 結果の検証（テスト実行・出力確認）
    ↓
完了（または次のステップへ）
```

任意の時点で **`Esc`** を押して中断し、別の指示を与えることができます。

---

## 基本的な使い方

### 会話スタイルで指示する

完璧なプロンプトは不要です。まず大まかに伝え、その後修正・追加します。

```
> このスクリプトのバグを直して

[Claude が調査・修正]

> セッション処理の部分も直して

[Claude がアプローチを修正]
```

### 具体的に伝えるほど精度が上がる

```
# 漠然とした例（動くが修正が多くなりがち）
> ログインを直して

# 具体的な例（一発で決まりやすい）
> src/auth/login.py のセッション処理でエラーが出ています。
  まずテストを書いてから修正してください。
```

### 中断と修正

| 操作 | 効果 |
|------|------|
| `Esc` | Claude を即座に停止。次の指示を入力できる |
| テキストを入力して `Enter` | 実行中のツールを止めずに追加指示を送る（Claude は現在のアクション完了後に読む） |

## よく使うコマンド

### セッション内スラッシュコマンド

| コマンド | 説明 |
|---------|------|
| `/help` | 利用可能なコマンド一覧を表示 |
| `/model` | 使用するモデルを変更 |
| `/compact` | コンテキストを圧縮してスペースを確保 |
| `/context` | コンテキストウィンドウの使用状況を確認 |
| `/resume` | 過去のセッションを選択して再開 |
| `/init` | プロジェクト用の `CLAUDE.md` を作成 |
| `/clear` | 会話履歴をクリアして新しいセッションを開始 |

### 起動時オプション

```bash
claude                        # 通常起動
claude --continue             # 直前のセッションをそのまま継続
claude --resume               # 過去のセッション一覧から選んで再開
claude --model claude-opus-4-8   # モデルを指定して起動
claude "ここにプロンプト"     # 一度だけ実行して終了（非対話モード）
```

## コンテキストウィンドウ

### コンテキストとは

Claude が「覚えていられる情報量」のことです。会話履歴・読み込んだファイルの内容・コマンドの出力などがすべてコンテキストウィンドウに積み上がっていきます。

```
┌────────────────────────────────────────┐
│ コンテキストウィンドウ                  │
│  ・会話履歴                            │
│  ・読み込んだファイルの内容            │
│  ・コマンドの実行結果                  │
│  ・CLAUDE.md の内容                    │
│  ・システム指示                        │
└────────────────────────────────────────┘
         ↑ 作業が進むと積み上がり、満杯になる
```

### 満杯になったらどうなるか

Claude は自動的に古い情報を圧縮（コンパクト化）します。ただし、会話の早い段階で与えた指示が失われることがあります。

**対策:** 重要なルールや設定は会話ではなく `CLAUDE.md` ファイルに書いておく。

### `/compact` コマンド

手動でコンテキストを圧縮します。長い作業セッションの途中でコンテキストが重くなってきたときに使います。

```bash
# シンプルな圧縮
/compact

# 重点を指定して圧縮（指定した内容を優先して保持）
/compact focus on the authentication changes
```

### `/context` コマンド

現在のコンテキスト使用量の内訳を表示します。何がスペースを消費しているか確認できます。

```bash
/context
```

---

## 会話のクリア（Clear）

`/clear` を使うと、現在の会話履歴をすべて削除して新しいセッションを開始します。コンテキストウィンドウが完全にリセットされるため、別のトピックや無関係な新しいタスクに切り替えるときに便利です。

```bash
/clear
```

> **`/compact` との違い:**
> - `/compact` は会話を**要約・圧縮**してコンテキストを軽くする（記憶は残る）
> - `/clear` は会話を**完全に削除**してゼロから始める（記憶はリセット）
>
> 同じ作業の続きには `/compact`、まったく別のタスクを始めるときは `/clear` を使います。

---

## セッションの再開（Resume）

Claude Code はすべての会話を `~/.claude/projects/` にローカル保存します。作業を中断してあとで続けることができます。

### 前回の続きをそのまま再開

```bash
claude --continue
```

### 過去のセッション一覧から選んで再開

```bash
claude --resume
# または起動後に
/resume
```

一覧が表示されるので、矢印キーで選択して `Enter` を押します。

> **注意:** 各セッションは独立しています。`--continue` や `--resume` を使わずに `claude` を起動すると、前のセッションの記憶のない新しい会話として始まります。

## モデルの選択

Claude Code は複数のモデルを切り替えて使えます。タスクの複雑さに応じて使い分けると効率的です。

| モデル | モデルID | 特徴 |
|--------|---------|------|
| **Claude Opus 4** | `claude-opus-4-8` | 最も高性能。複雑なアーキテクチャ設計・難しいバグ解析に |
| **Claude Sonnet 4** | `claude-sonnet-4-6` | バランス型。ほとんどのコーディングタスクに最適（**デフォルト推奨**） |
| **Claude Haiku 4** | `claude-haiku-4-5-20251001` | 高速・軽量。簡単な質問・繰り返しタスクに |

### モデルの変更方法

```bash
# 起動時に指定
claude --model claude-opus-4-8

# セッション中に変更
/model
```

`/model` を実行すると選択肢が表示されます。

---

## 権限モード

`Shift+Tab` を押すたびに権限モードが切り替わります。

| モード | 動作 |
|--------|------|
| **デフォルト** | ファイル編集・コマンド実行の前に毎回確認を求める |
| **自動受け入れ編集** | ファイル編集と基本的なファイル操作は確認なしで実行 |
| **Plan Mode** | 読み取り専用。実行前にプランを作成して確認できる |

初学者のうちは**デフォルト**モードが安全です。Claude が何をしようとしているか確認しながら進められます。

---

## CLAUDE.md — 永続的な指示ファイル

プロジェクトフォルダに `CLAUDE.md` を置くと、毎回のセッションで Claude が自動的に読み込みます。よく使うルールや注意点を書いておくと、毎回説明する手間が省けます。

```bash
# Claude に CLAUDE.md の作成を手伝ってもらう
/init
```

記述例：

```markdown
# CLAUDE.md

## プロジェクトの概要
GIS 解析の学習用リポジトリです。

## コーディング規約
- Python 3.12 を使用
- パッケージ管理は uv を使用
- コメントは日本語で書く
```

---

## 効果的に使うヒント

- **最初は具体的に:** ファイル名・エラーメッセージ・期待する動作を最初のプロンプトに含めると精度が上がる
- **検証材料を渡す:** テストケースやスクリーンショットを一緒に渡すと Claude が自分で結果を確認できる
- **実装前に探索させる:** Plan Mode（`Shift+Tab` 2回）で先にコードを読ませてから実装させると、的外れな変更が減る
- **長い作業は `/compact` で整理:** コンテキストが重くなってきたら `/compact` で整理してから続ける
- **`--continue` を習慣に:** 前回の続きを再開するときは `claude --continue` を使う
